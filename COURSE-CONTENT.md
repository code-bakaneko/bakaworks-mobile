# Course Content

Live contents of the `lesson_sets` tree, with room for your notes.

## How to leave notes

Write any line starting with `>` directly **underneath** an anchor. Anchors look
like `[item:42]`, `[set:3.2]` or `[lesson:3]` and never change.

```
- [item:42] listen — plays か → か / き / く / け
  > swap this for a harder distractor
  > and move it after the trace
```

Then run `node scripts/dump-content.js` — the file rebuilds from the database
and **your notes come back with it**, attached to the same anchors.

Notes on something that does not exist yet go in the Requests section at the
bottom, which is preserved as a whole.

##### あ &nbsp; `[lesson:1]`
_a_

**Set 1** &nbsp; `[set:1.1]`

- `[item:8]` **lecture** — Japanese is written with three systems at once. ひらがな and カタカナ are together called kana — each character is simply one sound, nothing more. 漢字 are characters borrowed from Chinese, and those carry meaning rather than sound. Hiragana is where everyone begins, because every Japanese word can be written in it.
- `[item:9]` **lecture** — The first character is あ. It is one sound — "ah", like the a in father, never the a in cake. It never changes: あ is あ in every word it appears in. Three strokes, always in the same order — the short horizontal, the vertical crossing it, then the loop.

## Set types

| type | meaning | content shape |
|------|---------|---------------|
| `lecture` | text to read; any Japanese in it is tappable for audio | `{text}` |
| `listen` | sound plays, learner picks the kana — no romaji | `{audio, choices, answer}` |
| `multiple_choice` | text prompt with text choices | `{prompt, choices, answer, audio?}` |
| `trace` | draw the character stroke by stroke | `{character, romaji, strokes[], viewBox}` |
| `video` / `audio` | declared, not implemented yet | `{url}` |
